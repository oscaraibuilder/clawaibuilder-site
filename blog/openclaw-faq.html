OpenClaw Setup FAQ

Honest answers to the questions we get most often. No marketing fluff.

1. What exactly is OpenClaw?

OpenClaw is an open-source AI agent framework. It runs on Node.js, connects to LLM providers like Anthropic, and gives you a persistent agent that can run tasks, read files, send messages, browse the web, and use tools — all on infrastructure you control. Think of it as the operating system for your AI agent. You own the VPS, you own the agent, you own the data. OpenClaw is what ties it together.

2. How long does a setup take?

Done yourself with no prior Linux/server experience: 2-3 days, realistically. There's a lot of configuration, and you'll hit problems that take hours to diagnose. Done yourself with a solid server background: 4-8 hours if you follow a good guide. Done for you via ClawAI Builder: 24 hours from payment to full handover with documentation.

3. What VPS do you recommend?

DigitalOcean Basic ($12/month, 2GB RAM, 2 vCPUs) for reliability. Hetzner CX22 (~$4.50/month, same specs) for price. Both run Ubuntu 22.04 LTS fine. Don't go below 2GB RAM — OpenClaw plus your model provider context handling eats memory, and you'll spend more time fighting swap than doing actual work. Avoid shared hosting; you need a real VPS.

4. Can you set it up on a Mac Mini?

Technically yes — OpenClaw runs on macOS. Practically, it's messier than a VPS. The main issue is that your agent goes offline whenever the Mac Mini restarts or loses power, and you don't have the same process management tools as Linux. If you have a Mac Mini that's always on with a stable connection, it works. But a $12/month VPS is more reliable, cheaper than electricity, and you don't have to think about it.

5. What does the memory system actually do?

Without a memory system, every OpenClaw session starts blank. The agent doesn't know your name, your preferences, what it did yesterday, or what's ongoing. The 3-layer system we set up gives the agent three levels of context: session notes (what happened in this conversation), daily logs (what happened today), and long-term MEMORY.md (everything important that's happened since setup). At the start of each session the agent reads these files and picks up exactly where it left off. It's the difference between a capable tool and an actual assistant.

6. Will my agent keep running when I close my laptop?

Yes — if it's running on a VPS. That's the whole point of the VPS setup. The agent runs on a server that's always on. You talk to it through Telegram from your phone or laptop — it doesn't matter whether your device is on or off. The agent's processes keep running on the server 24/7. If you're running it locally (on your laptop), then no, it stops when you close it.

7. What happens if my agent gets stuck in a loop?

It runs up your API bill. This is a real failure mode. A loop can burn $20-50 in an hour if you're on Claude Sonnet and it's calling the API repeatedly. The fix is a watchdog with spend limits — check the CostClaw skill on ClawMart for the setup. You also want PM2 managing your processes so the agent can be killed and restarted automatically when something goes wrong. We include loop detection and kill switches in the Operator and Full Stack setups.

8. Do I need to know how to code?

For the DIY setup: you don't need to write code, but you do need to be comfortable in a terminal. Creating files, editing config, running commands. If "SSH into a server" sounds scary, the done-for-you option will save you a lot of pain. If you're comfortable in a terminal but not a developer, the DIY setup is doable with a good guide. Full code knowledge helps but isn't required.

Three ways: check pm2 status on your VPS and it shows the process as "online"; send your agent a message on Telegram and it responds; or set up a cron job that sends you a daily ping at 8am (which is part of the Operator setup). If you're not getting the daily ping, something has gone wrong. The watchdog setup we include in Operator tier also sends a Telegram alert if the process dies unexpectedly.
